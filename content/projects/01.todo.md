Selamat hari ini dan Selamat kepada Gua yang telah menyelesaikan aplikasi TODO LIST Versi 1 (Kemungkinan akan dikembangkan lagi). Aplikasi ini dibuat karena gua sedang mempelajari Typscript dan NuxtJs, selain itu Aplikasi ini juga dibuat untuk kalian yang ingin melihat bagaimana kode-nya ditulis.

## Bagaimana Todo App Bekerja ?

Aplikasi ini bekerja dengan cara mencatat kegiatan yang ingin dilakukan, sehingga dengan sangat pasti memiliki indikator sudah dikerjakan atau belum dikerjakan, selain itu secara otomatis akan dibuatkan ID-Unique disetiap rencana kegiatan yang ingin dicatat, sehingga proses Update dan Remove kegiatan dapat diambil berdasarkan ID.

## Bagian Kode

Gua membuat 2 File yang berisikan sedikit kode agar aplikasi ini bisa berjalan sebagaimana mestinya, yang pertama adalah useTodo.ts yang berfungsi sebagai controller dan penyimpanan data reactive, lalu yang kedua adalah ProjectTodo.vue dengan tag `<script>` untuk mengakses controller, `<template>` untuk menampilkan apa yang perlu ditampilkan, dan terakhir `<style>` untuk memodifikasi tampilan agar lebih indah dan berwarna, akan tetapi, karena Gua menggunakan library Tailwind, maka tag `<style>` digunakan langsung di HTML dengan format tailwind.

Berikut adalah controller dari aplikasi TODO LIST

```ts [composables/useTodo.ts]
// Berfungsi sebagai model pada aplikasi
export interface Todo {
  // dengan interface Todo artinya aplikasi ini
  // akan memiliki properti sebagai berikut:
  id: number; // id akan selalu berbeda dengan kegiatan yang lainnya
  task: string; // nama bisa aja sama
  done: boolean; // untuk nandain apakah suatu kegiatan sudah dikerjakan atau belum
  popup: boolean; // sebuah toggle agar user bisa memilih kegiatan untuk di update, ditandai, dihapus
  createAt: Date; // ini sebagai informasi kapan kegiatan dibuat
  updatedAt: Date; // ini sebagai informasi kapan kegiatan diupdate
}

// core controller yang berfungsi untuk interaksi user terhadap
// kegiatan yang ingin disimpan ataupun tersimpan
export default function useTodo() {
  // tempat penyimpanan kegiatan
  // menggunakan reactive karena tampilannya akan berubah berdasarkan perubahan si todos,
  // sehingga pada bagian akses untuk controller tidak perlu melakukan reactive lagi
  const todos = reactive<Todo[]>([]);

  // fungsi untuk menambahkan kegiatan ke => const todos

  // untuk menambahkan kegiatan, Gua perlu mengambil id dan nama kegiatan yang ingin disimpan
  // karna sudah membuat interface, maka Gua berikan parameternya berdasarkan interface Todo
  // tetapi enggak Gua ambil semua, gua hanya memerlukan ID dan TASK
  const add = (todo: Pick<Todo, 'id' | 'task'>) => {
    // variable yang bertujuan untuk generate tanggal hari ini
    const now = new Date();

    // id dan kegiatan ditambahkan ke tempat penyimpanan (const todos),
    // Gua menambahkan default data untuk setiap kegiatan

    // setiap kegiatan, akan selalu diawali dengan belum dikerjakan (done: false)
    // setiap kegiatan popup-nya akan sembunyi dulu (popup:false)
    // ketika kegiatan dibuat, maka akan otomatis generate tangal kapan kegiatan dibuat (createAt: now)
    // karena baru di input maka updatedAt akan diisi dengan tanggal saat ini mengikuti createAt (updatedAt: now)
    todos.push({ ...todo, done: false, popup: false, createAt: now, updatedAt: now });
  };

  // fungsi untuk update kegiatan yang sudah ada atau dibuat //
  // (todo: Partial<Omit<Todo, 'id'>> & { id: number | null })
  // Gua membuat parameter yang harus di isi oleh user dari interface Todo (...<Todo,....)
  // akan tetapi gua ingin semua parameter bersifat opsional (Partial<)
  // semuanya opsional, kecuali id, karena memilih kegiatan yang akan di update berdasarkan id (id: number | null)
  const update = (todo: Partial<Omit<Todo, 'id'>> & { id: number | null }): boolean => {
    // dari tempat penyimpanan kegiatan (const todos) Gua akan mencari yang memiliki ID samadengan ID yang ingin di update
    const getTodoByID = todos.find((todoValue) => todoValue.id === todo.id);

    // jika sebuah pencarian tidak ditemukan maka fungsi update akan berakhir disini, dan tidak melakukan apa apa
    if (!getTodoByID) return false;

    // artinya yang dibawah ini adalah ketika sebuah pencarian berhasil menemukan ID yang di inginkan

    // ketika parameter task ditemukan, maka hasil pencarian akan merubah task (getTodoByID.task) dengan task yang baru (todo.task)
    if (todo.task !== undefined) {
      getTodoByID.task = todo.task;
    }

    // ketika parameter done ditemukan, maka hasil pencarian akan merubah done (getTodoByID.done) dengan done yang baru (todo.done)
    if (todo.done !== undefined) {
      getTodoByID.done = !todo.done;
    }

    // ketika parameter popup ditemukan, maka hasil pencarian akan merubah popup (getTodoByID.popup) dengan popup yang baru (todo.popup)
    if (todo.popup !== undefined) {
      getTodoByID.popup = !todo.popup;
    }

    // setiap melakukan update, maka tanggal dan waktu terbaru akan terupdate juga
    getTodoByID.updatedAt = new Date();

    return true;
  };

  // fungsi untuk menghapus kegiatan yang sudah ada

  // todo: Pick<Todo, 'id'> : Gua hanya mengambil parameter ID sebagai memilih kegiatan mana yang akan di hapus
  const remove = (todo: Pick<Todo, 'id'>): boolean => {
    // dikarenakan tempat penyimpanan memiliki fungsi reactive, maka melakukan penghapusan tidak bisa merubah secara langsung si const todos
    // maka untuk penghapusan menggunakan manipulasi array, yaitu dengan slice pada penyimpanan yang sudah ada.
    // dipilihlah fungsi findIndex untuk memilih index keberapa ID ditemukan pada sebuah penyimpanan
    const getIndexTodosByID: number = todos.findIndex((todoValue) => todoValue.id === todo.id);

    // ketika ID tidak ditemukan maka fungsi remove akan berhenti disini
    if (getIndexTodosByID === -1) return false;

    // ketika id ketemu, maka akan dilakukan penghapusan 1 kegiatan dari index yang ditemukan
    todos.splice(getIndexTodosByID, 1);

    return true;
  };

  // untuk menampilkan peyimpanan
  const read = (): Todo[] => todos;

  // akses yang di izinkan
  return {
    add,
    update,
    remove,
    read,
  };
}
```

Dari controller diatas, maka Gua bisa membuat user berinteraksi dengan penyimpanan

```vue [components/content/ProjectTodo.vue]
<script lang="ts" setup>
// Panggil si conteller dan ambil semua akses yang di izinkan
const { add, read, remove, update } = useTodo();

// ini adalah tipe data
interface RecordData {
  id: number;
  task: string;
  memo: number | null;
  isUpdate: Boolean;
  filter: string;
}

// membuat rekaman terhadap interaksi user dengan tipe pada interface RecordData
const record = reactive<RecordData>({
  id: 0,
  task: '',
  memo: null,
  isUpdate: false,
  filter: '',
});

// START untuk default kegiatan, bisa juga sebagai langkah untuk menambahkan kegiatan //
record.id += 1;
add({ id: 1, task: 'One' });

record.id += 1;
add({ id: 2, task: 'Two' });

record.id += 1;
add({ id: 3, task: 'Three' });

record.id += 1;
add({ id: 4, task: 'Four' });

record.id += 1;
add({ id: 5, task: 'Five' });
// END default kegiatan ... //

const getTodos = computed(() => {
  let result: Todo[] = read().sort((a, b) => b.id - a.id);

  switch (record.filter) {
    case 'date':
      return result.sort((a, b) => b.createAt.getTime() - a.createAt.getTime());
    case 'done':
      return result.sort((a, b) => Number(b.done) - Number(a.done));
    case 'not':
      return result.sort((a, b) => Number(a.done) - Number(b.done));
    default:
      return result;
  }
});

const handleNewOrUpdateTask = () => {
  if (record.isUpdate) {
    update({ id: record.memo, task: record.task });
    record.task = '';
    record.memo = null;
    record.isUpdate = false;
  } else {
    record.id += 1;
    add({ id: record.id, task: record.task });
    record.task = '';
  }
};
const updatePopup = (todo: Pick<Todo, 'id' | 'popup'>) => {
  update(todo);
};
const handleControlSelect = (todo: Pick<Todo, 'id' | 'done'>) => {
  update(todo);
};
const handleControlUpdate = (todo: Pick<Todo, 'id' | 'task'>) => {
  record.isUpdate = true;
  record.memo = todo.id;
  record.task = todo.task;
};
const handleControlDelete = (todo: Pick<Todo, 'id'>) => {
  remove(todo);
};
const taskTotal = computed<number>(() => read().length);

const taskDone = computed<number>(() => read().filter((item) => item.done).length);
const taskUndone = computed<number>(() => read().filter((item) => !item.done).length);
</script>
```

## Fitur Apa Saja yang disediakan ?

Berikut adalah fitur saat ini:

1. Buat kegiatan baru.
2. Update kegiatan yang sudah ada, seperti update nama tugasnya, atau update apakah kegiatan sudah selesai atau belum.
3. Menghapus kegiatan yang sudah ada.
4. Menampilkan kegiatan berdasarkan:
   - kegiatan yang sudah selesai.
   - kegiatan belum selesai.
   - tanggal terbaru.
   - tanggal paling lama.
5. User juga bisa memilih ingin menampilkan kegiatan selesai atau kegiatan belum selesai saja.
6. Terdapat box informasi terhadap jumlah kegiatan yang sudah dibuat dan jumlah kegiatan yang sudah selesai atau belum.
